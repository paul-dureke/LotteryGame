# ğŸ² Lottery Game

A console-based lottery game built with **Clean Architecture** principles, featuring configurable game rules, multiple prize tiers, and extensible design patterns.

## Features

- **Multi-Tier Prize System**: Grand Prize (50%), Second Tier (30%), Third Tier (10%)
- **Configurable Game Rules**: Customize prize percentages, player limits, and ticket ranges
- **Strategy Pattern**: Pluggable prize distribution algorithms
- **Clean Architecture**: Proper separation of concerns across Core, Business, and Presentation layers
- **Comprehensive Testing**: Full test coverage with mock providers

## Tech Stack

- **.NET 8.0**
- **C# 12.0**
- **xUnit** (Testing)
- **Clean Architecture Pattern**
- **Strategy Pattern**
- **Dependency Injection Ready**

## Quick Start

### Prerequisites
- .NET 8.0 SDK or later

### Run the Game
- git clone https://github.com/paul-dureke/LotteryGame.git
- cd LotteryGame 
- dotnet run

### Run Tests
dotnet test

## How It Works

1. **Player Setup**: Human player (Player1) + 10-15 CPU players
2. **Ticket Purchase**: Players buy tickets ($1 each, up to balance limit)
3. **Prize Drawing**: Three-tier system with configurable percentages
4. **Results Display**: Winners announced with prize amounts

## Configuration

Customize game behavior via `LotteryConfig`:
var customConfig = new LotteryConfig { DefaultPlayerBalance = 20, GrandPrize = { PrizePercentage = 0.6m }, MinPlayers = 5, MaxPlayers = 8 };
